        <h3>Ciclo de Vida do Software (SDLC)</h3>
        <p>Refere-se às etapas padrão para criar um produto de qualidade:</p>
        <ul>
            <li><strong>Requisitos:</strong> Definição do que o sistema deve fazer.</li>
            <li><strong>Design:</strong> Planejamento da arquitetura e interface.</li>
            <li><strong>Implementação:</strong> A fase de codificação propriamente dita.</li>
            <li><strong>Testes:</strong> Verificação de bugs e validação com o usuário.</li>
            <li><strong>Manutenção:</strong> Atualizações e correções pós-lançamento.</li>
        </ul>

    <section>
        <h2>2. Método Scrum</h2>
        <p>O Scrum é um <em>framework</em> ágil para gestão de projetos complexos, onde o foco é a entrega rápida e constante.</p>
        
        <h3>Papéis do Scrum</h3>
        <ul>
            <li><span class="highlight">Product Owner:</span> Representa os interesses do cliente e prioriza o backlog.</li>
            <li><span class="highlight">Scrum Master:</span> Facilitador que remove obstáculos e garante as regras do Scrum.</li>
            <li><span class="highlight">Time de Desenvolvimento:</span> Profissionais que executam o trabalho técnico.</li>
        </ul>

        <h3>Artefatos e Ritos</h3>
        <ul>
            <li><strong>Sprint:</strong> Ciclo de trabalho (geralmente 2 a 4 semanas).</li>
            <li><strong>Daily:</strong> Reunião diária para alinhamento do time.</li>
            <li><strong>Product Backlog:</strong> Lista de todas as tarefas desejadas para o produto.</li>
        </ul>
